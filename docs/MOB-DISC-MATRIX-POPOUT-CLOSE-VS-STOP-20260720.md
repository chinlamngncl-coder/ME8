# MOB DISC — Matrix popout: Close vs Stop

**Date:** 2026-07-20  
**Status:** LOCKED (operator meaning)  
**Related:** `POPOUT-CLOSE-SAFE-V1`, `POPOUT-MATRIX-FLV-READY-ACCEPT-V1`

---

## Plain English (yes — with one precision)

| Button | Meaning |
|--------|---------|
| **Close** | Close **only the matrix window**. Main desk wall panels and map pin **stay Live**. Like putting away Command Wall. |
| **Stop ■** on a matrix cell | Stop **that camera / that panel for real** — matrix cell **and** the matching ops wall panel (and pin if that cam). Not “hide popout only.” |

So:

- **Close** = dismiss the extra monitor window.  
- **Stop** = end live for **that** BWC on the desk, while the matrix is open (and after).

---

## Not “Stop all panels at once”

**Stop ■** is **per panel / per camera** in the matrix (one cell = one Stop).

- Stop Chin’s cell → Chin stops on wall + matrix.  
- kk’s other cell can stay Live until you Stop kk too.  
- There is **no** single matrix button that means “stop every panel in this popout” unless you click Stop on each cell (or we add a later MOB).

If you meant “Stop = stop that cam **everywhere** (wall + pin + popout)” → **Yes.**  
If you meant “one Stop kills every cam in the matrix” → **No** (today).

---

## Panel popout (`live.html`) — same idea

| Button | Meaning |
|--------|---------|
| **Close** | Close that one-cam window; wall + pin stay Live. |
| **Stop ■** | Stop that cam on the wall (real stop). |

---

## Locked product rule

**Close ≠ Stop.**  
Close never replaces Stop. Stop never means “only close the window.”

---

## One line

**Close = matrix away, desk keeps Live. Stop ■ = stop that panel’s cam on the desk (and matrix), not Close.**
