# MOB DISC — Zero change without APPLY / go-ahead

**Status:** LOCKED 2026-07-13 — user order  
**Search:** no creativity, no imagine, no change without apply, go ahead, discussion first  
**Supersedes soft wording:** any “tiny fix”, “obvious”, “while I’m here” edits

---

## Plain answer

**No creativity. No inventing. No imagining what might help.**

**Nothing** changes in the product (code, HTML, locales, bats, manuals, configs) unless:

1. We **discussed** it, and  
2. You said **`MOB-APPLY …`** or **go ahead** / **apply** for that exact thing.

Not even a **small** change. Not a one-line fix. Not “cleanup.” Not “while fixing X also Y.”

---

## Allowed without APPLY

| Allowed | Not allowed |
|---------|-------------|
| Read / search / diagnose | Edit any file |
| Answer questions | “Quick” brand / copy / IP / UI tweak |
| Write a MOB DISC **proposal** (paper only) | Apply the proposal |
| Run read-only checks | Kill processes / rewrite packs unless you ordered it |

---

## Agent must not

- Invent names, layouts, keys, or “better” wording  
- Decide C2 / Axiom / Alerts / health placement alone  
- Bundle extra fixes into a named MOB  
- Treat queue text **the agent wrote** as your approval  

---

## Order (only path)

```
Discuss (optional short Q)
  → you: MOB-APPLY <name>  OR  go ahead
  → agent changes ONLY that
  → stop
```
